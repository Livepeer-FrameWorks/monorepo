Before you fire up the CLI, let's make sure your environment is ready. FrameWorks has specific requirements for network topology and access control to function correctly.

## Hardware & OS Requirements

### Minimum Development Requirements
For testing or small-scale deployments, you can run a consolidated stack on a single node:
- **Single Node:** 4+ cores, 16GB RAM, 100GB SSD.
- **Note:** Performance will be limited for high-concurrency streaming or heavy analytics loads.

### Hardware (recommended)

**Central Tier:**
- CPU: 8+ cores
- RAM: 32GB+ (64GB recommended for YugabyteDB)
- Disk: 500GB+ SSD
- Network: 1Gbps+ connectivity

**Regional Tier:**
- CPU: 4+ cores
- RAM: 16GB+
- Disk: 250GB+ SSD
- Network: 1Gbps+ connectivity

**Edge Tier:**
- CPU: 8+ cores
- RAM: 16GB+
- Disk: 1TB+ SSD (for recordings)
- Network: 10Gbps+ connectivity (for high-bandwidth streaming)

### Required Software

**Docker Deployment:**
- Docker Engine (24.0+) and Docker Compose (2.20+) for containerized service deployment.

**Native Deployment:**
- `systemd` for service management.
- PostgreSQL client tools.
- Basic build tools (GCC, Make, Git) for native compilation.

**Both:**
- SSH server with key-based authentication.
- `curl`, `wget`, `tar`, `gzip`.
- `netstat` or `ss` for network diagnostics.

## Network Prerequisites

For multi-host deployments, you need:

1. **WireGuard Mesh Network** - Secure communication between all hosts. See [WireGuard Mesh Setup](/operators/wireguard) for configuration.

2. **DNS/Hosts Configuration** - Internal name resolution via `/etc/hosts` or Privateer's built-in DNS.

3. **Firewall Rules** - See [WireGuard Mesh Setup](/operators/wireguard#firewall-configuration) for required ports.

## SSH Access

### SSH Key Setup

Generate SSH keys for CLI access:

```bash
# Generate ED25519 key (recommended)
ssh-keygen -t ed25519 -C "frameworks-cli" -f ~/.ssh/frameworks_ed25519

# Or RSA key (4096-bit)
ssh-keygen -t rsa -b 4096 -C "frameworks-cli" -f ~/.ssh/frameworks_rsa
```

Copy public keys to all hosts:

```bash
ssh-copy-id -i ~/.ssh/frameworks_ed25519.pub admin@central-01
ssh-copy-id -i ~/.ssh/frameworks_ed25519.pub admin@central-02
# ... repeat for all hosts
```

### SSH Configuration

Configure `~/.ssh/config` for convenient access:

```
# Central Tier
Host central-01
    HostName 10.0.1.10
    User admin
    IdentityFile ~/.ssh/frameworks_ed25519

Host central-02
    HostName 10.0.1.11
    User admin
    IdentityFile ~/.ssh/frameworks_ed25519

# Regional Tier
Host regional-eu-01
    HostName 10.1.1.10
    User admin
    IdentityFile ~/.ssh/frameworks_ed25519

Host regional-us-01
    HostName 10.2.1.10
    User admin
    IdentityFile ~/.ssh/frameworks_ed25519

# Edge Nodes
Host edge-eu-01
    HostName 10.3.1.10
    User admin
    IdentityFile ~/.ssh/frameworks_ed25519
```

### User Permissions

The CLI user needs sudo privileges for:
- Installing packages
- Managing systemd services
- Configuring network settings
- Managing Docker (or be in docker group)

```bash
# Add user to docker group (Docker deployments)
sudo usermod -aG docker admin

# Or configure passwordless sudo for specific commands
admin ALL=(ALL) NOPASSWD: /usr/bin/systemctl, /usr/bin/docker
```

## External Services

FrameWorks requires several external services. See [External Services](/operators/external-services) for detailed setup guides.

**Required:**
- Cloudflare (DNS management required for Central/Regional tiers - *self-hosted Anycast DNS planned*)
- SSL/TLS certificates (ACME/Let's Encrypt managed via Navigator/Caddy)
- SMTP provider (email notifications)

**Optional:**
- Stripe (billing)
- MaxMind GeoIP (geographic routing)

## Next Steps

Once prerequisites are complete:

1. **Create Cluster Manifest:** See [Cluster Manifest Reference](/operators/cluster-manifest)
2. **Deploy Infrastructure:** `frameworks cluster provision --manifest cluster.yaml --components infrastructure`
3. **Initialize Databases:** `frameworks cluster init --manifest cluster.yaml`
4. **Deploy Applications:** `frameworks cluster provision --manifest cluster.yaml --components applications`
5. **Verify Health:** `frameworks cluster doctor --manifest cluster.yaml`

See [CLI Reference](/operators/cli-reference) for complete command documentation.
