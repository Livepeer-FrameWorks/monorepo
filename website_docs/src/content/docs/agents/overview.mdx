## Quick Start

1. **Create or load an EVM wallet.**
2. **Sign a wallet login message** (EIP-191) and call `POST /auth/wallet-login` to auto-provision a prepaid tenant.
3. **Check `account://status`** to confirm readiness and resolve any blockers.
4. **Fund the tenant** via x402 (`submit_payment`) or crypto deposit (`topup_balance`).
5. **Create a stream** via MCP `create_stream`, then push RTMP with the returned stream key.

<Aside type="tip">
  No email, no signup form, no API key. A wallet signature is all you need to get started.
</Aside>

## Discovery

FrameWorks publishes metadata following the [Agent Skills](https://agentskills.io) open standard so agents can discover and connect automatically.

| URL                                                                                           | Standard     | Purpose                                                                 |
| --------------------------------------------------------------------------------------------- | ------------ | ----------------------------------------------------------------------- |
| [`/.well-known/mcp.json`](%GATEWAY_URL%/.well-known/mcp.json)                                 | MCP          | Server discovery: endpoint, transports, auth schemes, public operations |
| [`/.well-known/did.json`](%GATEWAY_URL%/.well-known/did.json)                                 | W3C DID      | Decentralized identity and service endpoints                            |
| [`/.well-known/oauth-protected-resource`](%GATEWAY_URL%/.well-known/oauth-protected-resource) | RFC 8707     | OAuth resource metadata with wallet and x402 extensions                 |
| [`/.well-known/security.txt`](%GATEWAY_URL%/.well-known/security.txt)                         | RFC 9116     | Security contact and advisories                                         |
| [`/skill.json`](%MARKETING_URL%/skill.json)                                                   | Agent Skills | Machine-readable skill metadata (name, auth, endpoints, keywords)       |
| [`/SKILL.md`](%MARKETING_URL%/SKILL.md)                                                       | Agent Skills | Human/LLM-readable quick-start guide                                    |
| [`/llms.txt`](%MARKETING_URL%/llms.txt)                                                       | Emerging     | LLM-friendly documentation index                                        |
| `/robots.txt`                                                                                 | Standard     | Crawler directives (allows AI bots)                                     |

### Local Skill Installation

Install as a local skill for coding agents:

```bash
# Claude Code
mkdir -p ~/.claude/skills/frameworks-network
curl -o ~/.claude/skills/frameworks-network/SKILL.md %MARKETING_URL%/SKILL.md

# OpenClaw (via ClawHub)
clawhub install frameworks-network

# OpenClaw (manual)
mkdir -p ~/.openclaw/skills/frameworks-network
curl -o ~/.openclaw/skills/frameworks-network/SKILL.md %MARKETING_URL%/SKILL.md

# Gemini CLI / Cursor / other Agent Skills consumers
mkdir -p ./skills/frameworks-network
curl -o ./skills/frameworks-network/SKILL.md %MARKETING_URL%/SKILL.md
```

Once installed, the agent can activate the skill on demand to connect to and operate FrameWorks.

## Authentication Methods

Three methods, all stateless, no registration required:

| Method               | Best For                  | Details                                                                          |
| -------------------- | ------------------------- | -------------------------------------------------------------------------------- |
| **Wallet** (EIP-191) | Autonomous agents         | Sign a message, auto-provision a tenant. See [Wallet Auth](/agents/wallet-auth). |
| **x402** (EIP-3009)  | Pay-per-use agents        | Gasless USDC on Base/Arbitrum. See [Payments](/agents/payments).                 |
| **Bearer** (JWT)     | Dashboard users, API keys | From login or account settings.                                                  |
